# Implementation Plan: Sticky Table of Contents Component

## Goal

Add an automated, clickable Table of Contents (TOC) to all content/blog pages.

- Desktop: sticky, anchored to the left side of the page while the user scrolls
- Mobile: rendered inline, between the post title/H1 and the post body

---

## Plugins Required

### 1. `rehype-slug`
Adds `id` attributes to all headings in the rendered HTML (e.g. `<h2 id="what-is-a-nas">`). Required for anchor links to work.

### 2. `rehype-autolink-headings`
Wraps each heading in an `<a>` tag pointing to its own `id`. Makes headings themselves clickable anchor links.

### 3. Custom remark plugin: `remark-collect-headings`
A small custom plugin (written inline in `svelte.config.js` or as a file in `src/lib/`) that walks the markdown AST and collects all headings (depth, text, slug) into the file's `frontmatter` object. This is what feeds the `TOC.svelte` component with structured data at build/load time.

> `remark-toc` is NOT used here — that plugin injects a TOC into the markdown body itself, which is not what we want.

---

## Data Flow

```
.md file
  → remark-collect-headings plugin
      → extracts headings → writes to frontmatter.headings[]
  → rehype-slug
      → adds id="" to each heading in rendered HTML
  → rehype-autolink-headings
      → wraps each heading in an anchor link

+page.server.ts (load function)
  → calls getPostBySlug(slug)
  → metadata.headings[] is now available
  → passed to +page.svelte as `data.headings`

+page.svelte
  → passes headings[] to <TOC headings={data.headings} />
  → renders TOC.svelte in the correct position based on screen size
```

---

## Heading Data Shape

Each heading collected by the remark plugin:

```ts
interface Heading {
  depth: number      // 1 = h1, 2 = h2, 3 = h3, etc.
  text: string       // plain text of the heading
  slug: string       // the id that rehype-slug will generate (e.g. "what-is-a-nas")
}
```

Only include `depth` 2 and 3 (h2, h3) in the TOC — h1 is the post title, h4+ is too granular.

## TOC.svelte — Layout & Behavior

### Desktop
- Position: fixed or sticky, left side of the viewport
- Only visible when the post content is taller than the viewport
- Scrolls with page but stays in place (CSS `position: sticky`)
- Highlights the active heading as the user scrolls (uses `IntersectionObserver`)
- Hidden on mobile via CSS (`display: none` below breakpoint)

### Mobile
- Rendered inline in the page flow, between the H1 and the article body
- Collapsible (toggle open/closed) — default: collapsed
- Shown only on mobile via CSS

### Props

```ts
export let headings: Heading[]   // array from remark-collect-headings
```

### Active heading tracking
Use `IntersectionObserver` on the actual heading elements in the rendered article to detect which heading is currently in view, then highlight the corresponding TOC entry.

## Notes

- Skip headings with `depth === 1` (that's the post title, already shown separately)
- The slug generated by `rehype-slug` uses `github-slugger` under the hood — the custom remark plugin should use the same library to ensure slugs match exactly
- Install `github-slugger` as a dependency of the custom remark plugin: `bun add -D github-slugger`
- Consider skipping the TOC entirely (or hiding it) if a post has fewer than 3 headings
